Fixing the pop noise when resuming audio via headphones

If we have a media file and pause it, sometimes the powersaving feature in windows will put the soundcard to sleep. Upon resuming the file, the soundcard wakes up and there is a pop sound that is harmful to the ears. To fix this, we need to disable powersaving mode for the soundcard.

https://forums.lenovo.com/t5/Windows-8-1-8-7-Vista-and-XP/Win-7-Fixing-the-sound-card-power-save-speaker-click-on-the-n100/td-p/158176

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{4D36E96C-E325-11CE-BFC1-08002BE10318}\0000\PowerSettings]
"ConservationIdleTime"=hex:00,00,00,00
"PerformanceIdleTime"=hex:00,00,00,00
"IdlePowerState"=hex:00,00,00,00
"PM_EAPD_Mode"=hex:03

Making RipGrep work on Gvim

When we assign grepprg to rg, it is fast when executed directly from the CLI, but slow when run from inside GVIM. This is due to Windows Defender’s “Real-time protection” feature. If we disable this feature from the Settings menu, it gets enabled in the next restart. To disable this feature, we need to use the Group Policy Editor. This is initially disabled.

  1. Follow Method 2 in the link to enable “Group Policy Editor”

    https://www.itechtics.com/enable-gpedit-windows-10-home/

  2. Follow Method 1 in the link below to disable “Real-time protection” permanently.

    https://www.windowscentral.com/how-permanently-disable-windows-defender-antivirus-windows-10
